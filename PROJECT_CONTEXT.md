# Project Context: urin\n\n## プロジェクト概要\n(詳細決定後に追記)\n\n## 開発における絶対ルール (2026-04-08 承継)\nenja-studio での実績と事故防止の教訓に基づき、以下のルールを厳守する。\n\n### 1. 極小ステップ (Micro-Steps)\n- 一度のコード修正は原則10行以内、または1つの関数/コンポーネントに限定する。\n- 複数の論理的な変更を同時に行わない。\n\n### 2. 完全承認制\n- コードを書き換える前に、必ず「修正内容と影響範囲」を日本語で提示し、ユーザーの「OK」を待つ。\n\n### 3. Gitによる3分単位のバックアップ\n- 修正直前に `git add . && git commit -m "pre: [内容]"` \n- 修正完了後に `git add . && git commit -m "step: [内容]"` \n- 不具合時は即座に `git reset --hard HEAD` で復旧する。\n\n### 4. チャットログの物理保存と自動宣言\n- AIは全ての対話を `logs/transcripts/active_session.md` に即座に追記保存する。\n- セッション開始時に「チャット内容は確実に毎回保存しています」と宣言する。\n\n## セキュリティ防壁の設計図 (Security Architecture)\n制作初期段階から以下の防壁を優先的に実装する。\n\n1. **個人情報の秘匿性保護**: RLS (Row Level Security) 等による徹底隔離。\n2. **サーバーサイド認証の徹底**: ブラウザ側ではなく、API Routes/Server Actions で認証を確認。\n3. **機密情報の完全分離**: APIキー等を `.env.local` に集約し、ソースコードに直書きしない。\n4. **Web脅威への常時対策**: XSS (エスケープ処理), CSRF (認証ヘッダ), SQLインジェクションの防止。\n\n## ポート管理\n- **標準ポート**: 3001番 (enja-studio (3000) との同時起動を確保するため)
